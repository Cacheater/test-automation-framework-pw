# Instructions for GitHub Copilot

## Project Overview
Playwright Test Automation Framework with TypeScript, Page Object Model pattern, and GitHub Actions CI/CD.

## When Working with This Project

### Testing Guidelines
1. Always use the Page Object pattern located in `tests/pages/`
2. Tests are in `tests/e2e/` directory
3. Test data is in `tests/support/test-data.ts`
4. Use `LoginPage` class for login interactions
5. Tag tests with browser annotations: `@chrome`, `@firefox`, `@edge`, `@chrome-headless`, etc.

### File Structure Context
```
.github/workflows/playwright-tests.yml  <- GitHub Actions CI/CD
playwright.config.ts                    <- Playwright configuration
tests/
  e2e/login.spec.ts                    <- Test specs
  pages/login-page.ts                  <- Page objects
  support/
    test-data.ts                       <- Test data
    browser-matrix.ts                  <- Browser aliases
```

### Configuration Notes
- **Default URL**: https://www.globalsqa.com/angularJs-protractor/BankingProject/#/login
- **Timeouts**: 60s test timeout, 15s action timeout
- **Browsers**: Chrome, Firefox, Edge (headed and headless modes)
- **Reporters**: HTML, JUnit, Allure

### Common Tasks with GitHub MCP

**Create a test case issue**:
```
@github create issue title:"Add test for deposit functionality" 
```

**Create a branch for new feature**:
```
@github create branch feature/add-deposit-tests
```

**Create a PR for code review**:
```
@github create pull request title:"Add new deposit tests" head:"feature/add-deposit-tests" base:"main"
```

### Test Execution
- `npm test` - Run all tests
- `npm run test:headed` - Run with visible browser
- `npm run test:chrome` - Run Chrome only
- `npm run test:smoke` - Run smoke tests
- `npm run report` - View HTML report
- `npm run report:allure` - View Allure report

### CI/CD Pipeline
Tests run automatically on:
- Push to `main`, `master`, `develop`
- Pull request creation to those branches
- Artifacts uploaded: HTML report, JUnit, Allure results

## When Generating New Tests
- Follow the existing test structure in `login.spec.ts`
- Use `LoginPage` methods: `open()`, `clickCustomerLogin()`, `selectCustomer()`, `submit()`, `expectDashboardFor()`, `expectSubmitHidden()`
- Import from `@pages/login-page` and `@support/browser-matrix`
- Add test data to `tests/support/test-data.ts` if needed
- Include browser tags in test names: `@chrome`, `@firefox`, etc.

## Code Quality
- TypeScript strict mode enabled
- Path aliases configured: `@pages/*`, `@support/*`
- Prettier formatting on save
- Node modules: `node_modules/` excluded
